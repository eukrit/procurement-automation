8-Step Pipeline

1
Research • Manual + AI

Research & Prepare Vendor List

Source suppliers from Made-in-China, Alibaba, GlobalSources, ZoomInfo. Verify emails. Create a structured JSON with metadata, search criteria, and email verification notes.

data/{category}_suppliers.json
2
Template • Manual

Create Procurement Template

Define bilingual email body ({vendor_name}, {deadline}, {title} placeholders), extraction schema for Gemini to parse responses, and auto-reply context.

Schema sections: rates (price, MOQ, lead time) + product_specs (model, power, certs) + capabilities (OEM, branding)

scripts/seed_{category}_rfq.py Firestore: procurement_templates/{id}
3
Seed • Python Script

Seed Firestore

Run the seed script to create the template, inquiry, and vendor sub-documents. Also upserts each vendor into the master vendor_directory.

Creates: template → inquiry (with send_config, scoring_config, automation_config) → N vendors

python scripts/seed_{category}_rfq.py
rfq_inquiries/{id} rfq_inquiries/{id}/vendors/{vid} vendor_directory/{vid}
4
Gmail • Python Script

Create Gmail Label + Filter

Auto-label inbound replies from all vendor emails. Creates a nested label (e.g., Suppliers/EV Charger) and a filter matching all primary + alt contacts → apply label, never spam.

Requires DWD: gmail.labels + gmail.settings.basic

python scripts/setup_gmail_{category}_filter.py
5
Dispatch • Cloud Function

Send RFQ

Always dry-run first. The Cloud Function renders the bilingual email from the template, sends via Gmail DWD, logs the message in Firestore, and posts a Slack notification to the inquiry's channel.

POST /send-rfq {"inquiry_id":"...", "dry_run":true}
POST /send-rfq {"inquiry_id":"..."}
send-rfq (Gen2, Python 3.12) Gmail API Slack API
6
Dashboard • HTML

Create Dashboard Page

Public summary page on GitHub Pages: KPI cards, vendor shortlist with capabilities, automation timeline, infrastructure details. One page per RFQ round.

business-automation/docs/{slug}.html
https://eukrit.github.io/business-automation/{slug}.html
7
Monitor • Fully Automated

Automated Response Handling

Runs 24/7 without intervention. Gmail Watch detects replies via Pub/Sub, Gemini classifies intent and extracts structured data, auto-reply engine responds to vendor questions, Slack notifications keep humans in the loop.

8
Compare • MCP / REST

Compare Rates & Award

Side-by-side rate comparison using the inquiry's scoring_config weights. Normalizes across trade terms (FOB, CIF, DDP). Available via MCP tool or REST API.

MCP: compare_rates("RFQ-GO-...")
GET /procurement-mcp/api/inquiries/{id}/compare
